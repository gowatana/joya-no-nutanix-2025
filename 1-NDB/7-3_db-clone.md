# 7-3. クローンDBの作成

NDBでのクローンDBの作成でも、タイムマシンを利用します。
クローンDBを作成することで、容量を消費しないクローンや、定期的なリフレッシュのようなCDMを実現できます。

NDBでは、クローンDBは、クローン元のソースDBとは明確に区別されているため、それぞれで実行可能な操作も異なります。

NDBのクローンDBは、一緒に新規DBサーバー仮想マシンを作成するか、既存DBサーバー仮想マシンに作成することになります。
また、プリ / ポスト コマンドの実行機能を利用することで、データのマスキング（サニタイズ）なども実現できます。
